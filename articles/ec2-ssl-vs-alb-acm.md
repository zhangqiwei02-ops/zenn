# EC2 に証明書を置く前に、ALB + ACM を先に考えるべき理由

AWS で Web アプリを HTTPS 化したいとき、最初に思いつくのは「EC2 に SSL/TLS 証明書を入れる」という方法かもしれません。

ですが、AWS の設計として素直なのは、**Application Load Balancer（ALB）と AWS Certificate Manager（ACM）を組み合わせる構成** です。

この記事では、なぜ EC2 直置きより ALB + ACM が優先されやすいのかを、運用負荷・拡張性・試験での考え方の観点から整理します。

## 結論：HTTPS 化と運用簡素化が要件なら ALB + ACM を先に疑う

まず結論として、次の条件があるなら ALB + ACM が有力です。

- Web アプリを HTTPS 化したい
- 証明書管理を簡単にしたい
- 複数 EC2 に証明書を配布したくない
- 高可用性や Auto Scaling を考えたい

この場合、ALB に ACM 証明書を設定し、**ALB 側で TLS を終端する** 形がかなり自然です。

## なぜ EC2 直置きは不利になりやすいのか

EC2 に証明書を直接置く方法自体は可能です。
しかし、AWS らしい運用という観点では、管理コストが上がりやすいです。

たとえば、

- 各 EC2 で証明書設定が必要になる
- 更新タイミングの管理が面倒になる
- インスタンス数が増えるほど配布負荷が上がる
- Auto Scaling 環境ではさらに扱いづらい

といった問題が出ます。

つまり、**できるかどうか** ではなく、**長く運用しやすいか** で見ると EC2 直置きは不利になりやすいです。

## ACM の役割

AWS Certificate Manager は、SSL/TLS 証明書を発行・管理するためのサービスです。

ACM を使うと、対応サービスと連携しながら証明書管理を簡素化できます。
少なくとも「各 EC2 に手作業で証明書を入れる」よりは、かなり AWS らしい運用に寄せやすくなります。

## ALB の役割

Application Load Balancer は、HTTP/HTTPS レベルのリクエストを受けて、背後の EC2 へ振り分けるロードバランサーです。

ALB に ACM 証明書を設定すると、インターネットからの HTTPS リクエストを ALB が受け、そこで TLS を終端できます。

この形にすると、EC2 側は証明書運用からある程度切り離され、アプリケーション処理に集中しやすくなります。

## 比較表で整理する

| 観点 | EC2 に直接証明書を置く | ALB + ACM |
|---|---|---|
| 証明書管理 | 各 EC2 で必要 | ALB 側に集約しやすい |
| 台数増加への強さ | 弱くなりやすい | 比較的強い |
| Auto Scaling との相性 | 面倒が増える | 良い |
| TLS 終端 | EC2 側 | ALB 側 |
| 運用の王道感 | 低め | 高い |

この比較だけでも、AWS でどちらが好まれやすいかはかなり見えます。

## どんな要件で ALB + ACM が刺さるか

次のような要件が並んだら、ALB + ACM を先に考えるべきです。

- HTTPS 化したい
- 管理を簡単にしたい
- 高可用性が必要
- EC2 が複数台になる前提がある
- 将来的に Auto Scaling を使いたい

### 例えばどう考えるか
- **証明書を各インスタンスで抱える** → 運用が重くなる
- **ALB 側でまとめて受ける** → 管理が整理しやすい

この差は、実務ではじわじわ効きます。

## 試験での見分け方

AWS 認定試験では、問題文に次の要素が入ると ALB + ACM を選びやすくなります。

- HTTPS を有効にしたい
- 証明書管理を簡単にしたい
- 運用負荷を下げたい
- 高可用性を確保したい

逆に、EC2 側終端が必要な強い制約が書かれていないのに、わざわざ EC2 直置きを選ぶ理由はあまりありません。

## 実務ではどう使うか

典型的には、次のような構成になります。

Internet
  ↓
ALB + ACM
  ↓
EC2

この形にしておくと、

- 証明書更新の管理が整理しやすい
- EC2 の入れ替えが楽になる
- Auto Scaling との整合が取りやすい
- 責任分界が見えやすい

といったメリットがあります。

## まとめ

EC2 に証明書を直接置く方法はありますが、AWS で HTTPS 化の基本形を考えるなら、まずは **ALB + ACM** です。

- ACM で証明書を管理する
- ALB で HTTPS を受ける
- EC2 はアプリ処理に集中する

この切り分けを押さえておくと、試験でも実務でも判断がかなり安定します。
