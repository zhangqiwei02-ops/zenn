# CloudTrail・AWS Config・CloudWatch の違いを整理する

AWS の運用や監視を設計する際、**CloudTrail、AWS Config、Amazon CloudWatch** の役割を正しく理解しているかどうかで、設計の質が大きく変わります。

これらはすべて「監視」や「ログ」に関係するサービスですが、実際には同じものを見ているわけではありません。
この記事では、それぞれが **何を見ているサービスなのか** を軸に、役割の違いを整理します。

## 結論：見ている対象が違う

まず結論として、この3つは以下のように切り分けられます。

- **AWS CloudTrail**：AWS アカウント内の API 操作履歴を記録する
- **AWS Config**：AWS リソースの設定状態やその変化を追跡する
- **Amazon CloudWatch**：システムの性能・ログ・状態変化を監視する

同じ「監視系サービス」でも、見ているものはそれぞれ別です。

## AWS CloudTrail の役割

CloudTrail は、AWS 上で行われた API コールを記録するサービスです。
マネジメントコンソールでの操作も、CLI や SDK 経由の操作も、裏側では API として記録されます。

### 主な用途
- 監査対応
- セキュリティインシデントの調査
- 誰がどの操作を行ったかの追跡

たとえば「誰が EC2 インスタンスを削除したのか」「誰が IAM 設定を変えたのか」といった調査では、CloudTrail が中心になります。

## AWS Config の役割

AWS Config は、AWS リソースの設定状態を記録し、その変更履歴やルール準拠を評価するサービスです。

CloudTrail が「操作の履歴」を記録するのに対し、Config は「構成の状態」を見るサービスです。

### 主な用途
- セキュリティグループや IAM 設定の変更履歴の追跡
- 暗号化やタグ付けなどのルール準拠確認
- 構成の差分チェック

たとえば「公開してはいけない S3 バケットが存在していないか」「暗号化されていない EBS がないか」といった継続的評価には AWS Config が向いています。

## Amazon CloudWatch の役割

Amazon CloudWatch は、AWS ��境のメトリクスやログを収集・可視化し、アラームや通知を行うサービスです。

### 主な用途
- CPU 使用率やメモリ利用率の監視
- ALB や Lambda の異常検知
- CloudWatch Logs を使ったログ収集と検索
- アラームを起点にした自動アクション

CloudWatch は「誰が変更したか」や「設定がどう変わったか」ではなく、**システムが今どう動いているか** を見るためのサービスです。

## 比較して整理する

| サービス | 対象 | 役割 | 向いている問い |
|---|---|---|---|
| CloudTrail | API 操作 | 監査・証跡 | 誰が何をしたか？ |
| AWS Config | リソース設定 | 構成管理・ルール評価 | 設定は正しいか？ |
| CloudWatch | メトリクス・ログ | 性能監視・通知 | 今、正常に動いているか？ |

この表で整理すると、役割の違いがかなり見えやすくなります。

## 実務では競合ではなく補完関係

実務では、これらをどれか一つだけ選ぶことはあまりありません。
むしろ、次のように補完的に使うのが自然です。

1. **CloudTrail** で操作履歴を監査する
2. **AWS Config** で構成変更やルール違反を検知する
3. **CloudWatch** で稼働状況や異常を監視する

このように役割を分けることで、監査・統制・運用監視をバランスよく設計できます。

## 試験対策としての覚え方

AWS 認定試験では、これら3つを同時に並べて「どれを使うべきか」を問う問題がよく出ます。

判断の基準は単純です。

- **API 操作の追跡** → CloudTrail
- **設定や準拠状況の追跡** → AWS Config
- **性能やログの監視** → CloudWatch

この軸を持っておけば、かなり迷いにくくなります。

## まとめ

CloudTrail、AWS Config、CloudWatch は、どれも運用に欠かせないサービスですが、見ている対象は同じではありません。

- CloudTrail は **操作の証跡**
- AWS Config は **構成の状態**
- CloudWatch は **性能と稼働状況**

この切り分けを押さえておくと、実務でも試験でも判断がずっと楽になります。
