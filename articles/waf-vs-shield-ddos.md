# AWS WAF と AWS Shield の違いを整理する

AWS のセキュリティ関連サービスを学び始めると、AWS WAF と AWS Shield の役割が似て見えることがあります。

どちらも攻撃対策に関係するため、

- 何が違うのか
- どちらを選ぶべきなのか
- 一緒に使うものなのか

が曖昧になりやすいポイントです。

この記事では、AWS WAF と AWS Shield の違いを、レイヤー・用途・設計の観点から整理します。

## 結論

先に結論を書くと、両者は次のように役割が分かれています。

- **AWS WAF** は、HTTP/HTTPS リクエストに対する制御を行うサービス
- **AWS Shield** は、DDoS 攻撃に対する保護基盤を提供するサービス

そのため、両者は代替関係というより、**補完関係**にあります。

## AWS WAF の役割

AWS WAF は、Web アプリケーションの前段でリクエストを評価し、ルールに基づいて許可または拒否を行います。

代表的な用途は次の通りです。

- SQL インジェクションや XSS への対策
- 特定の国や IP レンジからのアクセス制限
- Rate-based ルールによる過剰アクセスの抑制
- 特定 URI やヘッダー条件に応じた制御

つまり AWS WAF は、**アプリケーション層での細かなフィルタリング**を担います。

## AWS Shield の役割

AWS Shield は、DDoS 攻撃への耐性を高めるためのサービスです。

AWS 上の公開サービスでは、アプリケーションのロジック以前に、大量の攻撃トラフィックに耐える必要があります。Shield は、この基盤側の防御を支えます。

### Shield Standard
基本的な DDoS 保護を提供します。主要な AWS サービスに対して広く適用される標準的な保護です。

### Shield Advanced
より高度な保護や運用支援が必要な場合に検討する上位オプションです。

たとえば、事業影響の大きいサービスや、DDoS リスクを強く意識すべき環境では有力な選択肢になります。

## 両者の違いを簡単に比較する

| 観点 | AWS WAF | AWS Shield |
|---|---|---|
| 主な対象 | HTTP/HTTPS リクエスト | DDoS 攻撃 |
| 主なレイヤー | L7 | 主に L3/L4 |
| 役割 | アプリケーション層の制御 | 基盤側の防御 |
| 設計上���位置づけ | 細かなルール設計 | 耐性の確保 |

このように見ると、同じ「セキュリティ対策」でも、見ている対象が異なることがわかります。

## 実務ではどう考えるべきか

実務では、AWS WAF と AWS Shield を片方だけで考えるより、**多層防御の一部として組み合わせる**方が自然です。

たとえば、次のような構成はよく見られます。

```text
Internet
  ↓
CloudFront (+ Shield)
  ↓
AWS WAF
  ↓
ALB / API Gateway
  ↓
Application
```

この構成では、

- Shield が基盤レベルの DDoS 耐性を担い
- WAF がアプリケーション層の制御を担う

という分担になります。

## よくある混同ポイント

### WAF があれば DDoS 対策として十分なのか
十分とは言えません。

WAF は重要な防御手段ですが、主にアプリケーション層の制御を担うものです。DDoS 対策全体を WAF だけで説明するのは不十分です。

### Shield があるなら WAF は不要なのか
これも違います。

Shield は基盤側の保護を担いますが、アプリケーションレベルでの細かなアクセス制御までは代替しません。

## どちらを選ぶべきか

問いの立て方としては、「どちらを選ぶか」よりも、**何を守りたいのか**を先に整理する方が適切です。

- HTTP/HTTPS リクエストを細かく制御したいなら AWS WAF
- DDoS 攻撃に対する耐性を高めたいなら AWS Shield
- 両方必要なら併用

この整理ができていれば、設計判断もしやすくなります。

## まとめ

AWS WAF と AWS Shield は、名前の印象ほど役割が重なっていません。

- AWS WAF は、アプリケーショ���層の制御
- AWS Shield は、DDoS に対する基盤防御

したがって、両者は競合するサービスではなく、**目的に応じて組み合わせて使うべきサービス**です。

セキュリティ設計では、単一のサービスに期待しすぎず、レイヤーごとに役割を分けて考えることが重要です。
